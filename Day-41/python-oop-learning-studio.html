Sheet 05 / Final Challenge

Build a Shape Hierarchy

Apply everything: classes, inheritance, encapsulation, polymorphism, and a dunder method — in one design task.

Challenge Brief

Design a base class Shape with a protected attribute for name, and an abstract-style method area() that raises NotImplementedError. Create subclasses Circle and Rectangle that override area(). Implement __str__ so print(shape) shows "Circle: 28.27"-style output. Then write a loop that computes total area of a list of mixed shapes polymorphically.

class Shape:
    def __init__(self, name):
        self._name = name

    def area(self):
        raise NotImplementedError

    def __str__(self):
        return f"{self._name}: {self.area():.2f}"

class Circle(Shape):
    def __init__(self, r):
        super().__init__("Circle")
        self.r = r
    def area(self):
        return 3.14159 * self.r ** 2

class Rectangle(Shape):
    def __init__(self, w, h):
        super().__init__("Rectangle")
        self.w, self.h = w, h
    def area(self):
        return self.w * self.h

shapes = [Circle(3), Rectangle(4,5)]
print(sum(s.area() for s in shapes))

Cheat Sheet

Class Basics

class Dog:
    species = "Canine"  # class attr
    def __init__(self, name):
        self.name = name  # instance attr
    def bark(self):
        return f"{self.name} woofs"

Encapsulation

self._protected   # convention
self.__private    # name-mangled
@property
def value(self): return self._v

Inheritance

class Puppy(Dog):
    def __init__(self, name):
        super().__init__(name)

__str__, __repr__

Summary Notes

  • Classes are blueprints; objects are instances built from them.
  • Encapsulation bundles data and behavior, restricting direct access via conventions and properties.
  • Inheritance lets a subclass reuse and extend a parent's structure via super().
  • Polymorphism lets different objects respond to the same method call in their own way.
  • Composition ("has-a") is often more flexible than deep inheritance chains ("is-a").
