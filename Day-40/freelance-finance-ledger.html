System prompt design

The assistant is scoped narrowly: budgeting, buffer funds, tax set-asides, income smoothing, and habit-building for people with irregular income — explicitly excluding stock picks, tax filing, and legal advice, with a standing disclaimer that it isn't a licensed advisor. This keeps every response inside the one job it's meant to do well.

Outputs are forced into a strict, delimited plain-text format (FLAG:, REC1_TITLE:, REC1_WHY:, REC1_ACTION:, ... HABIT:) rather than free-form prose or JSON. This is more robust to model drift than JSON (no bracket-matching failures) while still being trivially parseable with line-based regex on the frontend.

An explicit edge case is built in: if expenses exceed income even in the high month, or the buffer is critically thin, the model must populate FLAG with a direct, non-alarmist warning that gets rendered as a distinct red banner above the recommendations — so the most urgent issue is never buried in a numbered list.

UI decisions

The visual language borrows from a bookkeeper's ledger: ruled background lines, ink-navy and brass tones instead of a typical fintech green/gradient palette, and monospaced figures (IBM Plex Mono) for anything numeric, echoing how numbers are actually set in accounting. The hero waveform is the signature element — a jagged "before" line implying irregular income, sitting inside a card labelled unsteady → disciplined, tying the visual straight to the stated goal of the assistant (habit-building, not just a single fix).

Recommendations render as ledger line items (numbered, dashed rule between them) with a separate visually distinct "why" line and "action" box, so reasoning is never optional filler — it's structurally part of each entry, matching the requirement that every recommendation carry its reasoning.

Extending this assistant

  • Memory: persist each month's ledger entry (localStorage or a backend) so the assistant can compare this month's numbers to the last and flag trends, not just snapshots.
  • Tools: connect a bank-feed or CSV-import tool so expenses are pulled automatically instead of hand-entered, with the model reconciling and categorizing them.
  • Multi-step workflow: turn the single-session output into a quarterly check-in flow — first session sets a plan, later sessions review adherence and adjust recommendations accordingly.
  • Web search tool: for the tax set-aside recommendation, ground the suggested percentage in current self-employment tax brackets rather than a static rule of thumb.
