How the stop-check governs the loop

Planner

Reads the spec once and drafts an approach — key structures and edge cases — before any code exists.

Executor

Writes the first working draft from the plan. Runs once; every later draft comes from the Improver.

Evaluator

Scores the current draft 0–10 on readability, best practices, and structure — fresh, every round.

Critic

Reads the score and the code to name specific, fixable issues the number alone can't convey.

Safety Monitor

Checks for unsafe patterns. An unresolved flag overrides the threshold stop — quality alone can't end the loop unsafely.

Improver

Rewrites the code using this round's evaluation, critique, and the running memory note. Feeds the next Evaluator call.

Memory Manager

Condenses the round's history into a short note so recurring issues aren't relearned from scratch each pass.

Final Reviewer

Runs once, only after plateau, threshold, or the hard cap fires, and writes the closing verdict.

Architecture overview

        
Planner → Executor produce the seed draft. Evaluator ⇄ Critic ⇄ Improver form the real loop — Improver's output feeds straight back into the next Evaluator call, each one a live API request carrying forward score, critique, and memory. Safety Monitor gates every round; Memory Manager compresses history so context doesn't grow unbounded. The loop exits only through plateau, threshold, or hard-cap, then hands off once to Final Reviewer.

      

    


    

      

        
Extension ideas

        
    
          
  • Add a Test-Writer agent that generates unit tests alongside code, feeding pass/fail back into the Evaluator's rubric.
    • 
          
  • Persist memory notes across sessions (e.g. localStorage substitute via a server) so the pipeline learns your team's style over time.
    • 
          
  • Run Executor variants in parallel (different temperatures) and let a Selector agent pick the strongest seed draft.
    • 
          
  • Swap the rubric per language — a TypeScript-specific rubric could weight type-safety alongside readability.
