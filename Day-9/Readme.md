## What I Did

For Day 9, I built NutriScope, an AI-powered nutrition analysis application using an iterative AI development approach.

Instead of requesting a complete advanced system in one prompt, I followed a structured workflow:

###  Step 1: MVP Development
* Started with a simple prompt to generate a working version of NutriScope
* Built a basic HTML application
* Added core functionality: food input and simple nutrition output
* Ensured the application was functional and stable

###  Step 2: Iterative Enhancement
* Used a second prompt to improve the MVP
* Added UI improvements and better layout
* Enhanced user interaction and output formatting
* Improved responsiveness and design consistency

### Step 3: Comparison & Analysis
* Compared MVP vs Enhanced version
* Captured screenshots of both versions
* Identified improvements in usability, design, and structure

---

## Key Learnings
* Building large apps in a single prompt reduces quality and control
* MVP-first approach improves stability and clarity
* Iterative development helps refine features step-by-step
* AI tools work best when guided in small, focused instructions
* Comparing versions helps understand real improvement impact
* Professional AI builders always iterate, never overbuild in one step
